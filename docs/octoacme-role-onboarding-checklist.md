# OctoAcme Role Onboarding Checklist

## Purpose
Provide a practical, role-specific checklist to help new team members quickly onboard, understand their responsibilities, and engage effectively with the team and project processes.

## How to use this checklist
1. Select the role section that matches your position
2. Work through the checklist items systematically
3. Check off items as you complete them
4. Reach out to your Project Manager or Scrum Master if you have questions

---

## All Roles - General Onboarding

### Week 1: Getting Started
- [ ] Complete company/team onboarding and access setup
- [ ] Review [octoacme-project-management-overview.md](./octoacme-project-management-overview.md)
- [ ] Review [octoacme-roles-and-personas.md](./octoacme-roles-and-personas.md)
- [ ] Meet with manager or Project Manager for role expectations
- [ ] Get added to relevant communication channels (Slack, Teams, etc.)
- [ ] Access project boards, repositories, and documentation
- [ ] Attend team standup or sync meeting

### Week 2: Understanding Context
- [ ] Review current project charter or one-pager
- [ ] Review project backlog and upcoming milestones
- [ ] Shadow a team ceremony (planning, review, or retrospective)
- [ ] Identify and introduce yourself to key collaborators
- [ ] Review relevant process docs for current project phase

### Ongoing
- [ ] Ask questions and seek feedback regularly
- [ ] Document learnings and share with the team
- [ ] Participate actively in team ceremonies and discussions

## Role-Specific Onboarding

### Developers

#### Technical Setup
- [ ] Clone repositories and set up local development environment
- [ ] Review coding standards and contribution guidelines
- [ ] Set up CI/CD access and understand build/test pipelines
- [ ] Review architecture documentation and technical design docs
- [ ] Complete first small task or bug fix to validate setup

#### Collaboration
- [ ] Pair with another Developer on a feature or code review
- [ ] Understand Definition of Done and acceptance criteria process
- [ ] Learn branching strategy and PR workflow
- [ ] Review test coverage expectations and testing approach
- [ ] Identify Technical Architect or tech lead for questions

---

### Product Managers

#### Context Building
- [ ] Review product vision, roadmap, and OKRs
- [ ] Understand customer personas and user research findings
- [ ] Review success metrics and how they're measured
- [ ] Understand competitive landscape and market positioning
- [ ] Meet with key stakeholders and understand their needs

#### Process Engagement
- [ ] Review backlog prioritization approach
- [ ] Understand sprint/iteration planning process
- [ ] Learn how to write user stories and acceptance criteria
- [ ] Identify cadence for stakeholder updates and reviews
- [ ] Shadow or lead a backlog refinement session

---

### Project Managers

#### Project Context
- [ ] Review project charter, timeline, and key milestones
- [ ] Understand current risks, dependencies, and blockers
- [ ] Review resource allocation and team composition
- [ ] Access and understand project tracking tools
- [ ] Review communication plan and stakeholder matrix

#### Process Ownership
- [ ] Schedule or attend project kickoff (if not yet held)
- [ ] Set up regular status reporting and stakeholder updates
- [ ] Review and update risk register
- [ ] Facilitate first team ceremony (standup, planning, etc.)
- [ ] Establish meeting cadence with Product Manager and tech leads

---

### Business Analysts

#### Requirements Context
- [ ] Review existing requirements documentation and user stories
- [ ] Understand business domain and key business processes
- [ ] Learn requirements traceability approach
- [ ] Identify key stakeholders and subject matter experts
- [ ] Review acceptance criteria standards

#### Collaboration Setup
- [ ] Schedule requirements workshops or elicitation sessions
- [ ] Partner with Product Manager on backlog refinement
- [ ] Establish working relationship with Developers and QA
- [ ] Review tools for process mapping and documentation
- [ ] Participate in planning or design discussion

---

### UX/UI Designers

#### Design Context
- [ ] Review existing design system, style guides, and brand standards
- [ ] Understand user personas and user research approach
- [ ] Review accessibility standards and compliance requirements
- [ ] Access design tools (Figma, Sketch, Adobe XD, etc.)
- [ ] Review any existing user flows or wireframes

#### Collaboration Setup
- [ ] Meet with Product Manager to understand product vision
- [ ] Establish handoff process with Developers
- [ ] Understand design review and feedback process
- [ ] Identify opportunities for user research or usability testing
- [ ] Participate in planning session to understand upcoming features

---

### Technical Architects

#### Architecture Context
- [ ] Review system architecture diagrams and documentation
- [ ] Understand technology stack, frameworks, and libraries
- [ ] Review architectural decision records (ADRs)
- [ ] Understand security, scalability, and performance requirements
- [ ] Review code standards and design patterns in use

#### Technical Leadership
- [ ] Conduct architecture review session with development team
- [ ] Identify technical debt and improvement opportunities
- [ ] Establish consultation process for technical decisions
- [ ] Participate in planning and estimation discussions
- [ ] Document any new architectural decisions or patterns

---

### Scrum Masters

#### Team Understanding
- [ ] Understand team composition, skills, and dynamics
- [ ] Review team velocity and historical sprint performance
- [ ] Understand Definition of Done and team working agreements
- [ ] Learn about current blockers and impediments
- [ ] Review retrospective action items and follow-through

#### Facilitation Responsibilities
- [ ] Facilitate or shadow daily standup
- [ ] Prepare for and facilitate sprint planning
- [ ] Schedule and facilitate sprint review and retrospective
- [ ] Establish backlog refinement cadence
- [ ] Set up one-on-ones with team members for coaching

---

### Support/DevOps Engineers

#### Infrastructure Context
- [ ] Access production and non-production environments
- [ ] Review infrastructure architecture and diagrams
- [ ] Understand CI/CD pipeline configuration and tools
- [ ] Review monitoring, logging, and alerting setup
- [ ] Learn incident response and escalation procedures

#### Operational Responsibilities
- [ ] Review deployment schedule and release process
- [ ] Understand rollback and disaster recovery procedures
- [ ] Access runbooks and operational documentation
- [ ] Review SLOs, SLAs, and reliability metrics
- [ ] Participate in or shadow a deployment
- [ ] Join on-call rotation (if applicable)
